## 1. Контекст

Рассматривается задача сортировки элементов целочисленного массива с применением алгоритма Шелла и последующего простого слияния отсортированных фрагментов. 

Последовательная реализация выполняет роль:
- эталона для верификации корректности вычислений,
- базовой метрики (baseline) для оценки ускорения и масштабируемости в параллельных версиях алгоритма.

---

## 2. Постановка задачи

Необходимо выполнить сортировку элементов входного массива (вектора) по неубыванию. Размерность входных данных может быть произвольной.

### Условия корректности
- Результирующий массив должен представлять собой перестановку элементов исходного массива, упорядоченную по неубыванию.
- Алгоритм должен корректно обрабатывать граничные случаи (пустые массивы и массивы, состоящие из одного элемента).

---

## 3. Базовый алгоритм

В основе решения лежит парадигма «разделяй и властвуй». Исходная структура данных логически разделяется на две части:

### Фаза сортировки (Shell Sort)
- Массив разбивается на левый и правый подмассивы.
- Каждый подмассив сортируется независимо с применением классического алгоритма Шелла. Данный метод является оптимизированным вариантом сортировки вставками, позволяющим сравнивать и обменивать элементы, находящиеся на значительном удалении друг от друга.

### Фаза слияния (Merge)
- Отсортированные части объединяются в итоговый массив с использованием функции слияния (`Merge`).
- Процедура слияния требует выделения дополнительной памяти, соразмерной объему сливаемых данных, и выполняется за один последовательный проход.

### Вычислительная сложность
- Время выполнения: O(N log^2 N) для фазы сортировки Шелла. Фаза простого слияния выполняется строго за время O(N).
- Пространственная сложность: O(N), что обусловлено необходимостью аллокации памяти для временных векторов (левой и правой частей) и результирующего массива.

## 4. Детали реализации

Файлы:
- `seq/include/ops_seq.hpp`
- `seq/src/ops_seq.cpp`

Архитектура пайплайна:
- **ValidationImpl**
  - Осуществляется проверка входной структуры на пустоту.
- **PreProcessingImpl**
  - Выполняется глубокое копирование входного вектора в структуру для вывода с целью последующей сортировки in-place (на месте).
- **RunImpl**
  - Инициируется вызов основной вычислительной процедуры `ShellSortMerge`.
- **PostProcessingImpl**
  - Не требует дополнительных действий, так как финальный результат формируется на этапе `RunImpl`.

### Особенности
- Вектор строго делится на две равные (или отличающиеся на один элемент) части.
- Использование вспомогательных векторов `left` и `right` увеличивает объем потребляемой памяти, однако гарантирует тривиальность и стабильность алгоритма слияния.

## Приложение

Листинг демонстрирует базовое ядро алгоритма (`ShellSortMerge`). Исходный массив разделяется пополам, каждая из половин сортируется в независимом контексте, после чего происходит вызов процедуры линейного слияния.

```cpp
void LitvyakovDShellSortSEQ::ShellSortMerge(std::vector<int> &vec) {
  if (vec.size() <= 1) {
    return;
  }
  std::size_t mid = vec.size() / 2;
  std::vector<int> left(vec.begin(), vec.begin() + static_cast<int64_t>(mid));
  std::vector<int> right(vec.begin() + static_cast<int64_t>(mid), vec.end());
  BaseShellSort(left);
  BaseShellSort(right);
  Merge(left, right, vec);
}
```
