## 1. Контекст

Рассматривается задача сортировки массива целых чисел с использованием алгоритма сортировки Шелла и последующего простого слияния. 

Последовательная реализация используется как:
- эталон корректности,
- база для сравнения ускорений в параллельных версиях.

---

## 2. Постановка задачи

Необходимо отсортировать входной массив (вектор) целых чисел по возрастанию. 
Размер массива может быть произвольным. 

### Условия корректности
- Результирующий массив должен содержать те же элементы, что и исходный, упорядоченные по неубыванию.
- Корректная обработка пустых массивов и массивов из одного элемента.

---

## 3. Базовый алгоритм

Используется алгоритм, разбивающий исходный вектор на две половины:

### Фаза сортировки (Shell Sort)
- Массив делится на левую и правую части.
- Каждая часть сортируется независимо с использованием классической сортировки Шелла (Shell Sort). Сортировка Шелла является обобщением сортировки вставками, которая позволяет обменивать элементы, находящиеся далеко друг от друга.

### Фаза слияния (Merge)
- Отсортированные левая и правая части сливаются в один отсортированный массив с использованием функции слияния (`Merge`).
- Слияние происходит с выделением дополнительной памяти под итоговый вектор и последовательным проходом по элементам обеих частей.

### Асимптотика
- Время: O(N log^2 N) в сортировке Шелла. Фаза слияния занимает строго O(N).
- Память: O(N) на выделение дополнительных векторов для левой и правой части, а также результирующего массива.

## 4. Детали реализации

Файлы:
- `seq/include/ops_seq.hpp`
- `seq/src/ops_seq.cpp`

Pipeline реализации:
- **ValidationImpl**
  - проверка входного вектора на пустоту.
- **PreProcessingImpl**
  - копирование входного вектора в выходной для последующей сортировки in-place.
- **RunImpl**
  - вызов функции `ShellSortMerge`.
- **PostProcessingImpl**
  - результат уже сформирован внутри `RunImpl`.

### Особенности
- Вектор строго делится на две половины.
- Используются дополнительные векторы `left` и `right`, что увеличивает потребление памяти, но упрощает алгоритм слияния.

## Приложение

Фрагмент показывает базовое ядро алгоритма (`ShellSortMerge`). Массив делится пополам, каждая половина сортируется независимо, после чего вызывается функция слияния.

void LitvyakovDShellSortSEQ::ShellSortMerge(std::vector<int> &vec) {
  if (vec.size() <= 1) {
    return;
  }
  std::size_t mid = vec.size() / 2;
  std::vector<int> left(vec.begin(), vec.begin() + static_cast<int64_t>(mid));
  std::vector<int> right(vec.begin() + static_cast<int64_t>(mid), vec.end());
  BaseShellSort(left);
  BaseShellSort(right);
  Merge(left, right, vec);
}
