## 1. Контекст

Отчет посвящен анализу масштабируемости алгоритма с использованием библиотеки Intel OneTBB (Threading Building Blocks), являющейся 
индустриальным стандартом для распараллеливания C++ приложений на системах с общей памятью с использованием концепции задач (task-based 
parallelism).

---

## 2. Постановка задачи

Необходимо реализовать многопоточную обработку массивов, делегировав управление балансировкой вычислительной нагрузки внутреннему планировщику TBB.

---

## 3. Базовый алгоритм и Архитектура

Основа остается неизменной (двухфазная схема с предварительным вычислением границ блоков). Различие заключается в механике распределения.

Вместо жесткой привязки потока к конкретной итерации (как в STL или OMP `schedule(static)`), TBB использует парадигму графов задач и 
алгоритм "кражи работы" (work-stealing). Свободные логические ядра могут динамически заимствовать необработанные задачи из очередей 
занятых ядер, нивелируя дисбаланс, возникающий на уровне операционной системы.

---

## 4. Схема распараллеливания

### Область параллелизма
Ключевым примитивом выступает шаблон `tbb::parallel_for`, оперирующий абстракцией пространства итераций `tbb::blocked_range<std::size_t>`.

```cpp
tbb::parallel_for(tbb::blocked_range<std::size_t>(0, parts_count), [&](const tbb::blocked_range<std::size_t> &range) {
  for (std::size_t i = range.begin(); i != range.end(); ++i) {
    const std::size_t l = bounds[i];
    const std::size_t r = bounds[i + 1];
    BaseShellSort(vec.begin() + static_cast<std::ptrdiff_t>(l), vec.begin() + static_cast<std::ptrdiff_t>(r));
  }
});
```

Политика разделения (Partitioner) не указывается явно, делегируя принятие решений эвристике Auto Partitioner. Синхронизация между 
задачами не требуется благодаря изолированному адресному пространству каждого подмассива.
