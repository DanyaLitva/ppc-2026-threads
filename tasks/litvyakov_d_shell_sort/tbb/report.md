## 1. Контекст

Рассматривается параллельная реализация алгоритма сортировки Шелла с использованием библиотеки Intel OneTBB.

Цель:
- уменьшить время выполнения сортировки
- применить примитивы TBB для управления задачами

---

## 2. Постановка задачи

Необходимо отсортировать массив целых чисел in-place с минимальным выделением дополнительной памяти.

### Условия корректности
- Результирующий массив должен быть упорядочен.
- Корректная обработка размеров массивов, не кратных числу потоков.
- Сравнение производительности с SEQ-версией.

---

## 3. Базовый алгоритм

Сохраняется та же двухфазная логика:
1. Вычисление границ кусков массива (функция `GetBounds`).
2. Параллельная независимая сортировка каждого куска с использованием алгоритма Шелла.
3. Последовательное слияние отсортированных фрагментов.

Разница заключается исключительно в технологии распараллеливания первой фазы.

---

## 4. Схема распараллеливания

### Область параллелизма
Параллелизуется независимая сортировка. Выбранный примитив — `tbb::parallel_for`. В качестве диапазона итераций используется `tbb::blocked_range<std::size_t>`. Библиотека TBB автоматически балансирует нагрузку между рабочими потоками (work-stealing).

tbb::parallel_for(tbb::blocked_range<std::size_t>(0, parts_count), [&](const tbb::blocked_range<std::size_t> &range) {
  for (std::size_t i = range.begin(); i != range.end(); ++i) {
    const std::size_t l = bounds[i];
    const std::size_t r = bounds[i + 1];
    BaseShellSort(vec.begin() + static_cast<std::ptrdiff_t>(l), vec.begin() + static_cast<std::ptrdiff_t>(r));
  }
});

Partitioner явно не задается, используется поведение oneTBB по умолчанию (Auto Partitioner).

## 5. Детали реализации

Файлы:
- `tbb/include/ops_tbb.hpp`
- `tbb/src/ops_tbb.cpp`

Pipeline реализации аналогичен OMP-версии, но используется лямбда-выражение внутри `parallel_for` для описания тела задачи. Гонки данных отсутствуют, так как границы `l` и `r` не пересекаются для разных индексов `i`. Фаза слияния использует `std::inplace_merge` и работает в один поток.

## 8. Результаты

Параметры эксперимента:
- Размер массива: 2 000 000 элементов  

Результат измерения (baseline SEQ = 0.059871 с):

| workers | time (s) | speedup | efficiency |
| -------- | --------- | -------- | ------------ |
| 2 | 0.030432 | 1.97 | 0.98 |
| 4 | 0.017436 | 3.43 | 0.86 |
| 8 | 0.013644 | 4.39 | 0.55 |

Наблюдения:
- Библиотека TBB показывает лучшие результаты производительности по сравнению с OpenMP, достигая ускорения 4.39х на 8 потоках.
- Эффективность на 2 и 4 потоках держится на высочайшем уровне (0.98 и 0.86 соответственно).
- Динамический балансировщик OneTBB (work-stealing) отлично справляется с задачей, минимизируя накладные расходы при планировании блоков.

- TBB успешно применён для распараллеливания независимых задач сортировки.

- Использование `tbb::parallel_for` позволило написать безопасный, производительный и масштабируемый код без явного управления потоками.
