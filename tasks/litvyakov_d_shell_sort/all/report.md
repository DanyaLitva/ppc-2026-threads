## 1. Контекст

Рассматривается гибридная параллельная реализация алгоритма сортировки Шелла, использующая:
- распределённую память (MPI)
- общую память (OpenMP)

Цель:
- создание реализации, эффективно использующей совместные вычисления на множестве узлов/процессов (MPI) и внутриядерной многопоточности (OpenMP).

---

## 2. Постановка задачи

Необходимо отсортировать массив целых чисел.

### Условия корректности
- Размер входного массива известен только корневому процессу (rank 0).
- Итоговый отсортированный массив должен быть корректно собран на нулевом процессе и затем разослан всем процессам для успешного прохождения автоматизированных тестов.

---

## 3. Алгоритм (MPI уровень)

На уровне процессов алгоритм состоит из следующих этапов:

1. **Broadcast размера массива**
   - Нулевой процесс рассылает всем процессам размер входного массива $n$ (`MPI_Bcast`).

2. **Разбиение данных (Scatter)**
   - Вычисляются массивы смещений `displs` и количества элементов `sendcounts` для каждого процесса (учитывается равномерное распределение остатка).
   - Нулевой процесс распределяет части исходного массива всем рабочим процессам с помощью `MPI_Scatterv`.

3. **Локальная сортировка**
   - Каждый процесс сортирует полученный локальный фрагмент `local_vec` при помощи функции `ShellSortMerge`. 

4. **Сбор результата (Gather)**
   - Отсортированные локальные фрагменты собираются обратно на нулевом процессе с использованием `MPI_Gatherv`.

5. **Итоговое слияние**
   - Нулевой процесс последовательно объединяет (`std::inplace_merge`) полученные от процессов отсортированные блоки.

6. **Broadcast результата**
   - Для валидации результатов всеми процессами в инфраструктуре тестов отсортированный массив пересылается всем узлам (`MPI_Bcast`).

---

## 4. Алгоритм (OpenMP уровень)

Внутри каждого процесса при сортировке локального массива (`ShellSortMerge`) применяется распараллеливание на уровне потоков:
- Локальный массив делится на блоки.
- Блоки сортируются при помощи `BaseShellSort` внутри директивы `#pragma omp parallel for`.
- Отсортированные блоки локального фрагмента сливаются `std::inplace_merge` внутри данного процесса.

---

## 5. Схема распараллеливания и синхронизации

### MPI уровень
- Процессы независимы.
- Обмен данными строго синхронизирован коллективными вызовами MPI: `MPI_Bcast`, `MPI_Scatterv`, `MPI_Gatherv`.

### OpenMP уровень
- Используется общая память процесса. Данные разделены неявно через выделение уникальных непересекающихся диапазонов итераций цикла.

---

## 6. Детали реализации

Файлы:
- `all/include/ops_all.hpp`
- `all/src/ops_all.cpp`

Pipeline:
- **ValidationImpl**: проверяется непустота вектора (на нулевом процессе).
- **PreProcessingImpl**: входные данные подготавливаются для распределения.
- **RunImpl**:
  - `MPI_Scatterv` частей массива.
  - локальный гибридный вызов `ShellSortMerge` (OpenMP).
  - `MPI_Gatherv` отсортированных частей.
  - Финальное слияние кусков на `rank == 0`.
- **PostProcessingImpl**: 
  - `MPI_Bcast` размера и самого массива.

Где ShellSortMerge:
  #pragma omp parallel for default(none) shared(vec, bounds, parts_count_t) schedule(static)

    const std::size_t r = bounds[i + 1];
    BaseShellSort(vec.begin() + static_cast<std::ptrdiff_t>(l), vec.begin() + static_cast<std::ptrdiff_t>(r));
  }
