## 1. Контекст

Данный отчет посвящен реализации параллельной схемы вычислений с использованием стандартной библиотеки потоков C++ (`std::thread`). Разработка этой версии служит аналитическим инструментом для оценки накладных расходов ручного распределения вычислительных задач (по паттерну `create -> work -> join`) в сравнении со специализированными фреймворками.

---

## 2. Постановка задачи

Требуется разработать потокобезопасную инфраструктуру для распределения блоков массива между физическими (или логическими) ядрами процессора средствами стандартной библиотеки C++, гарантируя детерминированность результатов и отсутствие гонок данных.

---

## 3. Базовый алгоритм

Архитектура сохраняет концепцию декомпозиции, реализованную в версиях OMP и TBB:
1. Вычисление оптимального количества пулов задач (`parts_count`).
2. Определение точных смещений (индексов) подмассивов.
3. Инстанцирование потоков, каждому из которых делегируется обработка выделенного сегмента памяти.
4. Синхронизация потоков и последовательная редукция результатов в главном потоке.

---

## 4. Схема распараллеливания

### Область параллелизма и модель управления потоками
Распараллеливание осуществляется явной аллокацией системных ресурсов через создание пула потоков: `std::vector<std::thread> thread_pool(parts_count)`.

1. Планирование задач реализуется через передачу контекста (в виде захвата переменных в лямбда-выражение) каждому инстансу `std::thread`.
2. Операционная система (планировщик ОС) берет на себя ответственность за маппинг созданных потоков на аппаратные ресурсы.
3. Главный поток (Main thread) переходит в режим ожидания, последовательно вызывая метод `.join()` для каждого рабочего потока.

### Управление памятью и синхронизация
- Глобальная память вектора передается по ссылке. Конфликты доступа полностью исключены математически: индексы `l` и `r` формируют строгие изолированные подмножества.
- Явный барьер синхронизации реализован циклом ожидания `join()`. Использование объектов-мьютексов не требуется, что исключает блокировки (lock-free) на фазе вычислений.

## Приложение

Листинг явного ручного управления пулом потоков и синхронизацией:

```cpp
  std::vector<std::thread> thread_pool(parts_count);

  for (std::size_t i = 0; i < parts_count; ++i) {
    const std::size_t l = bounds[i];
    const std::size_t r = bounds[i + 1];
    thread_pool[i] = std::thread([&vec, l, r]() {
      BaseShellSort(vec.begin() + static_cast<std::ptrdiff_t>(l), vec.begin() + static_cast<std::ptrdiff_t>(r));
    });
  }

  for (auto &t : thread_pool) {
    t.join();
  }
```
