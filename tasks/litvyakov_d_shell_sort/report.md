## 1. Введение

В данной работе исследованы несколько реализаций алгоритма сортировки массива (сортировка Шелла с использованием простого слияния). Рассматривались реализации:
- Без использования технологий параллелизма (последовательная версия, технология **SEQ**);
- С использованием библиотеки OpenMP (технология **OMP**);
- С использованием библиотеки Intel OneTBB (технология **TBB**);
- С использованием стандартной библиотеки `std::thread` (технология **STL**);
- С использованием MPI для параллелизма на уровне процессов совместно с OpenMP для потоков (технология **ALL**).

Сортировка хорошо поддается распараллеливанию по принципу "разделяй и властвуй". Исходный массив дробится на блоки, каждый поток/процесс сортирует свой блок независимо алгоритмом Шелла, после чего отсортированные блоки сливаются.

---

## 2. Единая постановка задачи

Необходимо отсортировать входной массив целых чисел по возрастанию in-place с минимальным использованием дополнительной памяти.

В ходе выполнения задачи требовалось:
1. Реализовать последовательную версию данного алгоритма.
2. Реализовать параллельные версии с использованием технологий OMP, TBB, STL и ALL.
3. Проверить корректность реализаций и провести экспериментальные замеры.
4. Сравнить время работы параллельных реализаций с последовательной реализацией.

---

## 3. Единая методика эксперимента

### Среда
- CPU: AMD Ryzen 5 5600
- RAM: 48 GB
- OS: Windows 10
- Компилятор: clang++21

### Тестовые данные
- Для `performance`-теста генерировался вектор из **2 000 000** псевдослучайных целых чисел.
- Для `functional`-тестов использовались массивы размерностью 1, 10, 100 и 1000 элементов.

### Используемые метрики
- Time (время) - медиана запусков.
- Speedup (ускорение) - отношение времени последовательной реализации к времени сравниваемой реализации.
- Efficiency (эффективность) = speedup / workers.

## 4. Сводка корректности

Все задачи сравнивались с SEQ-реализацией в отношении корректности.
Проверка выполнялась общей инфраструктурой:
- `functional`-тесты и `performance`-тесты.

В технологии `ALL` для обеспечения успешного прохождения тестов всеми процессами, итоговый результат в `PostProcessingImpl` рассылался всем процессам при помощи `MPI_Bcast`. Все тесты успешно пройдены.

## 6. Интерпретация результатов

`SEQ` является базисом.

Фреймворки с общей памятью (`TBB`, `STL`) демонстрируют практически идентичную и очень высокую производительность (ускорение ~4.3x на 8 потоках). Ручная реализация через `std::thread` на данной задаче не уступила балансировщику TBB. Технология `OMP` показала себя несколько хуже (ускорение 3.75x).

Ограничением для всех технологий с общей памятью является алгоритмическое финальное последовательное слияние блоков (`std::inplace_merge`).

Гибридная технология `ALL` показывает относительно небольшой прирост относительно OMP при конфигурациях 2x1 и 2x2 (ускорение ~2.8x против 2.56x).

## 7. Заключение

Все реализации алгоритма "Сортировка Шелла с простым слиянием" успешно имплементированы и протестированы. Использование многопоточности существенно сокращает время работы независимой сортировки подмассивов, обеспечивая пиковое ускорение до **4.39x**. Для решения задачи на одной машине целесообразнее всего использовать `TBB` или чистый `STL`.
